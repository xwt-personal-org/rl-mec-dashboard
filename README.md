# RL-MEC Training Dashboard

轻量级网页看板，用于实时监控 RL-MEC Benchmark 训练任务状态。Dashboard 只读训练输出文件，不启动、不停止、不重启 paper2 训练任务。

## 功能

- 实时状态监控：优先读取 `experiments/<run_id>/state.json`，展示当前算法、状态、日志和结果。
- 多运行总览：固定显示 Full 17 与 Quick 入口，并兼容 legacy logs、structured runs 和 mixed runs。
- paper2 实验协议优先：优先读取 `experiments/<run_id>/run.json`、`state.json` 和每算法 artifacts。
- Legacy fallback：没有 paper2 experiments 时继续读取旧 `runs/`、`logs/benchmark*.log` 和 `results/benchmark.json`。
- 多实验对比：`/api/compare` 支持按 reward、latency、energy 等指标对比。
- Convergence 曲线：从 `benchmark*.json` 的 `convergence_by_seed` 懒加载 reward-step 数据，支持当前 run 与 backup/archive，并可点击算法切换。
- 结果导出：支持 CSV 和 Markdown 表格导出。
- SSE 实时推送：前端通过 Server-Sent Events 刷新 run snapshot。
- Stop Dashboard Server：只关闭 dashboard server，不停止训练任务。

## 启动方式

### Windows 菜单栏一键启动（推荐）

首次安装快捷方式：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\install_start_menu_shortcut.ps1
```

安装后从 Windows 开始菜单启动：



说明：此入口只启动 dashboard server，不启动、不停止、不重启 paper2 训练任务。

## 数据源优先级

1. paper2 实时状态：`experiments/<run_id>/run.json` + `experiments/<run_id>/state.json`
2. 每算法结果：`experiments/<run_id>/artifacts/<ALGORITHM>/result.json`
3. benchmark 图表导出：`results/benchmark_<run_id>.json`
4. convergence 曲线：`results/benchmark_<run_id>.json` 或 backup/archive 内 `benchmark*.json` 的 `convergence_by_seed`
5. legacy fallback：旧 `runs/`、`logs/`、`results/benchmark.json`

## Convergence curves

The Convergence panel does not display static PNG files generated by `paper2`. It lazy-loads curve data and renders an interactive Chart.js line chart.
- Primary source: `convergence_by_seed` in `results/benchmark_<run_id>.json` or backup/archive `benchmark*.json`.
- Fallback source: `experiments/<run_id>/artifacts/<ALGORITHM>/checkpoints/train_logs.json` for active runs and experiment-backed backups.
- Default metric is `reward`; API also supports `latency`, `energy`, and `comm_score`.
- Train-log fallback maps `eval_eval/*` keys onto the same API shape and spreads points across `train_timesteps` when `run.json` provides it.
- Archive-only backups without `convergence_by_seed` still show no data because they do not contain experiment artifacts.


Backup visibility APIs:

- `GET /api/backups`
- `GET /api/runs/{run_id}/backups`
- `GET /api/backups/diagnostics`

Danger Zone deletion APIs:

- `GET /api/delete-targets`
- `POST /api/delete-preview`
- `POST /api/delete-confirm`

Deletion is permanent from the dashboard perspective. The browser sends only `target_id`, preview returns exact paths and a confirm token, running experiments are blocked, and dashboard does not provide restore/fresh/start/stop training actions.
